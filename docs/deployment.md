## 1. Prerequisites

- **MongoDB Atlas** cluster (free tier works)
- **Cloudinary** account (for resume/logo uploads)
- **Google Gemini API key**
- **ElevenLabs API key** (avatar TTS)
- **OpenAI API key** (Whisper transcription)
- **n8n** instance (Cloud or self-hosted Docker)
- **Gmail OAuth2 + Google Calendar OAuth2 + Google Sheets OAuth2** credentials configured in n8n

---

## 2. Deploying Backend Services (Render)

### 2.1 Using the Blueprint

The project includes a `render.yaml` blueprint at the repo root. To use it:

1. Push the repo to GitHub.
2. Go to [Render Dashboard вҶ’ Blueprints](https://dashboard.render.com/blueprints).
3. Connect your GitHub repo.
4. Render will detect `render.yaml` and provision 3 services automatically:
   - `ai-interview-server` (Node.js)
   - `ai-interview-python-ai` (Python)
   - `ai-interview-avatar-service` (Node.js)

### 2.2 Environment Variables

After the blueprint deploys, fill in the `sync: false` variables in the Render dashboard:

#### Server (`ai-interview-server`)

| Variable | Value |
|----------|-------|
| `MONGODB_URI` | Your MongoDB Atlas connection string |
| `GEMINI_API_KEY` | Google Gemini API key |
| `CLIENT_URL` | Your Vercel deployment URL (e.g. `https://aurarecruit.vercel.app`) |
| `PYTHON_AI_URL` | Render URL of Python AI service (e.g. `https://ai-interview-python-ai.onrender.com`) |
| `AVATAR_SERVICE_URL` | Render URL of Avatar service (e.g. `https://ai-interview-avatar-service.onrender.com`) |
| `CLOUDINARY_CLOUD_NAME` | Cloudinary cloud name |
| `CLOUDINARY_API_KEY` | Cloudinary API key |
| `CLOUDINARY_API_SECRET` | Cloudinary API secret |
| `N8N_WEBHOOK_REGISTRATION` | `https://<n8n-host>/webhook/registration` |
| `N8N_WEBHOOK_RESUME_REJECTED` | `https://<n8n-host>/webhook/resume-rejected` |
| `N8N_WEBHOOK_INTERVIEW_SCHEDULED` | `https://<n8n-host>/webhook/interview-scheduled` |
| `N8N_WEBHOOK_INTERVIEW_REMINDER` | `https://<n8n-host>/webhook/interview-reminder` |
| `N8N_WEBHOOK_INTERVIEW_COMPLETE` | `https://<n8n-host>/webhook/interview-complete` |
| `N8N_WEBHOOK_OFFER` | `https://<n8n-host>/webhook/offer` |

> `JWT_SECRET` is auto-generated by the blueprint.

#### Python AI (`ai-interview-python-ai`)

| Variable | Value |
|----------|-------|
| `GEMINI_API_KEY` | Google Gemini API key |
| `SERVER_URL` | Render URL of server service |

#### Avatar Service (`ai-interview-avatar-service`)

| Variable | Value |
|----------|-------|
| `SERVER_URL` | Render URL of server service |
| `GEMINI_API_KEY` | Google Gemini API key |
| `ELEVEN_LABS_API_KEY` | ElevenLabs API key |
| `OPENAI_API_KEY` | OpenAI API key (for Whisper) |
| `CLIENT_URL` | Vercel deployment URL |

## 3. Deploying the Client (Vercel)

### 3.1 Setup

1. Go to [vercel.com](https://vercel.com) and import the GitHub repo.
2. Set the **Root Directory** to `client`.
3. Set the **Framework Preset** to `Vite`.
4. Add environment variable:
   - `VITE_API_URL` вҶ’ Render URL of your server (e.g. `https://ai-interview-server.onrender.com/api`)

### 3.2 Security Headers

`vercel.json` ships with production security headers:
- `X-Content-Type-Options: nosniff`
- `X-Frame-Options: DENY`
- `X-XSS-Protection: 1; mode=block`
- `Referrer-Policy: strict-origin-when-cross-origin`
- `Permissions-Policy: camera=(), microphone=(self), geolocation=()`

Static assets under `/assets/` are cached with `immutable, max-age=31536000`.

## 4. CORS Configuration

CORS is enforced on all 3 backend services:

| Service | Allowed Origins |
|---------|----------------|
| Server | `CLIENT_URL` env var + `http://localhost:5173` |
| Python AI | `SERVER_URL` + `CLIENT_URL` + `http://localhost:5173` |
| Avatar Service | `CLIENT_URL` + `http://localhost:5173` |

**In production**, set `CLIENT_URL` to your exact Vercel domain (e.g. `https://aurarecruit.vercel.app`). The localhost fallback is only active when no `CLIENT_URL` is set.

## 5. n8n Workflow Setup

### 5.1 Hosting n8n

**Option A: Docker (recommended)**
```bash
docker run -d --name n8n \
  -p 5678:5678 \
  -e N8N_BASIC_AUTH_ACTIVE=true \
  -e N8N_BASIC_AUTH_USER=admin \
  -e N8N_BASIC_AUTH_PASSWORD=<strong-password> \
  -v n8n_data:/home/node/.n8n \
  n8nio/n8n
```

**Option B: n8n Cloud** вҖ” [n8n.io/cloud](https://n8n.io/cloud)

### 5.2 Importing Workflows

1. Open n8n UI at `http://<host>:5678`.
2. For each file in the `n8n/` directory, go to **Workflows вҶ’ Import from File** and select the JSON.
3. Configure the required credentials on each node:
   - **Gmail OAuth2** вҖ” all 5 webhook workflows with email nodes
   - **Google Calendar OAuth2** вҖ” `interview_scheduled.json`
   - **Google Sheets OAuth2** вҖ” `interview_complete.json`

### 5.3 Workflow Summary

| Workflow | Trigger | What It Does |
|----------|---------|-------------|
| Registration | Webhook | Sends welcome email on signup |
| Resume Rejected | Webhook | Sends improvement feedback email with weaknesses & recommendations |
| Interview Scheduled | Webhook | Sends invite email вҶ’ creates Calendar event вҶ’ calls back `/interview/:id/calendar-synced` |
| Interview Reminder | Webhook | Sends reminder email on HTTP POST request |
| Interview Complete | Webhook | Appends to Google Sheets вҶ’ emails HR with evaluation summary |
| Offer Email | Webhook | Sends official offer letter to candidate |

### 5.4 Production URLs

Update the HTTP Request node in `interview_scheduled.json` to use your deployed Render server URL instead of `http://localhost:5000`.

## 7. Monitoring & Logs

- **Render**: Each service has real-time logs in the Render dashboard.
- **Server**: Winston logger outputs structured logs. Set `NODE_ENV=production` for JSON log format.
- **n8n**: Execution history is available in the n8n UI under each workflow.
